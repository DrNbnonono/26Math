1.1 评委的分数怎么来的
变量构造（从CSV直接算）
对选手$$i$$在周 $$t$$：
- 评委总分：$$S_{i,t} = \sum_j \text{score}_{i,t,j}$$
- 为了消除每周评委人数不同（3或4），用评委均分： $$J_{i,t} = \frac{S_{i,t}}{\#\text{judges}_t}$$
推荐模型：分层/混合效应线性模型（最合适）
$$J_{i,t} = \beta_0 + \beta_{\text{age}} Age_i + \beta_{\text{ind}}[Industry_i] + \gamma_{\text{pro}}[Pro_i] + \delta_{\text{season}}[Season_i] + \delta_{\text{week}}[Week_t] + \epsilon_{i,t}$$
解释：
- $$\beta_{\text{age}}$$: 年龄对评委得分的影响
- $$\beta_{\text{ind}}$$: 行业对评委得分的影响（运动员/演员/歌手等）
- $$\gamma_{\text{pro}}$$: 职业舞者的“教学/编舞/带飞”能力差异（核心！）
- season/week项控制“不同季、不同周整体给分尺度”变化

1.2 观众的投票怎么来的
- 第一问输出每周投票占比$$\hat p_{i,t}$$或票数$$\hat v_{i,t}$$。观众票是比例数据，使用占比来进行分析：
用 logit 变换后做混合效应回归：
$$\text{logit}(\hat p_{i,t})=\alpha_0+\alpha_{age}Age_i+\alpha_{ind}[Industry_i]+\eta_{pro}[Pro_i]+\alpha_{home}[Home]+\alpha_{season}[Season]+\alpha_{week}[Week_t]+u_i+\varepsilon_{i,t}$$
解释：
- $$\eta_{pro}$$：职业舞者的“人气/粉丝盘/镜头表现”是否会带来额外票
- 行业、家乡、国家更可能强烈影响投票（比评委分更敏感）
- 我觉得种族可能也挺影响的

AI说这俩模型挺互补的，
模型互补性强：
- 序数逻辑回归：这是一个参数模型，假设累积概率（cumulative logit），适合解释特征影响（e.g., “年龄每增加1岁，排名变差的odds ratio是1.2”）。它线性、简单，但对非线性关系（如年龄对fan votes的U形影响）捕捉不足。
- 梯度提升序数模型（e.g., XGBoost或LightGBM的ordinal loss扩展）：这是一个非参数树模型，能自动捕捉交互（e.g., industry * ballroom_partner）和非线性。它预测准确高，但解释性稍差（需SHAP补）。
- 融合两者：能取长补短。序数逻辑提供基线解释，梯度提升提升精度，尤其在你的数据中（n≈400，特征多如industry one-hot），融合能让模型更鲁棒。
