我来逐一说明所有生成的文件及其用途：

## 一、代码文件（Python脚本）

| 文件 | 用途 |
|------|------|
| [1-preprocess.py]| **数据预处理脚本**：读取原始CSV，处理缺失值，计算评委总分和占比，识别淘汰信息 |
| [1-model-percentage.py]| **百分比法模型**：对第3-27季建立约束优化模型，估算观众投票比例 |
| [1-model-rank.py]| **排名法模型**：对第1-2季、28-34季建立排名优化模型 |
| [1-visualizations.py] | **可视化脚本**：生成5张论文所需的图表 |
| [1-generate-final-results.py]| **结果整合脚本**：整合所有结果，生成最终的3个缺失文件 |

## 为什么输出这么多文件？

按照数学建模竞赛的规范要求，需要：
1. **可复现性**：代码文件让别人能重复你的分析
2. **过程透明**：中间结果文件便于检查每一步的正确性
3. **结果完整**：最终文件满足题目要求的"1-XXXX"命名格式
4. **论文支撑**：图表和Tex文件用于撰写正式论文

**核心交付物**就3个：最终的投票估算CSV、一致性指标JSON、论文Tex文件。其他都是支撑材料。
